┌──────────────────────────────────────────────────────────────────────────────┐
│ THE DOUBLE-LOOP ARCHITECTURE                                                │
└──────────────────────────────────────────────────────────────────────────────┘

     RESEARCH               PLANNING              EXECUTION + VERIFY
         │                    │                          │
         ▼                    ▼                          ▼
    Gemini Deep          Claude Code              CodeWhale (Inner)
    Research            plan mode                 • Edit files
         │                    │                   • LSP diagnostics
         ▼                    ▼                   • Fix errors
    Domain Doc          Tripartite Plan           • Loop until LSP=∅
         │                    │                          │
         ▼                    ▼                          ▼
   HITL Gate 1         HITL Gate 2            Claude Code (Outer)
   Approve/Steer       Approve + escalate     • Run exit clauses
   Reject/Restart      permission             • Verify architecture
         │                    │                • Review security
         ▼                    ▼                • Check contracts
    RESEARCH_RUN      PLANNING_RUN                     │
    GATE1_PENDING     GATE2_PENDING           • PASS: promote snapshot
                                              • FAIL: restore + redispatch
                                                      │
                      EXECUTION_RUNNING ◄─────────────┘
                              │
                              ▼
                        Repeat until
                        all modules
                        PASSED
                              │
                              ▼
                      FINALIZATION_RUN
                      (Linter, tester,
                       builder agents)
                              │
                              ▼
                          COMPLETED ✓
                       + Report + Cost

KEY GUARANTEE:
Deterministic exit clauses run BEFORE the supervisor.
The supervisor can veto a green module, but CANNOT override a failed clause.
